# SKOS Version Policy

Version: 0.0.1

---

## Philosophy

Every change must be traceable.

Every release must be reproducible.

Nothing is anonymous.

---

## Project Version

Format

MAJOR.MINOR.PATCH

Example

0.0.1

0.1.0

1.0.0

---

## Build Number

Every completed build receives a permanent Build ID.

Example

BUILD-000001

BUILD-000002

BUILD-000003

...

Build IDs are never reused.

---

## Sprint

Development is organized by Sprint.

Example

Sprint 01

Sprint 02

Sprint 03

...

---

## Release

Public releases use

Release-0.1.0

Release-1.0.0

Release-2.0.0

---

## Repository Rule

Every completed BUILD requires exactly one Commit.

Every Sprint contains one or more BUILDs.

Every Release contains one or more Sprints.

---

## Traceability

Every document should reference:

Version

Sprint

Build

Date
