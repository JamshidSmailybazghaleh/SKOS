# SKOS Repository Rulebook

Version: 1.0.0

---

# Core Principle

Architecture before implementation.

Standards before development.

Knowledge before code.

---

# Repository Rules

## Rule 01

Every BUILD must produce a measurable output.

---

## Rule 02

Every BUILD requires exactly one Commit.

---

## Rule 03

Every Commit must reference its BUILD number.

---

## Rule 04

Every source file must contain the official SKOS Header.

---

## Rule 05

Every document must contain:

Version

Build

Date

Status

---

## Rule 06

No undocumented source code.

---

## Rule 07

No anonymous Knowledge Object.

---

## Rule 08

No duplicate is permanently deleted.

Relationships are preferred over deletion.

---

## Rule 09

Standards have priority over implementation.

---

## Rule 10

Repository structure must remain stable.

---

# Engineering Principle

Build once.

Improve forever.

Never rebuild without reason.

---

# Knowledge Principle

Knowledge is preserved.

Knowledge is traceable.

Knowledge is expandable.

Knowledge is never orphaned.

---

# Project Motto

Useful Knowledge Never Dies.
